What is PruneVision?

A real-world AI application built on top of the self-pruning neural network case study

01

The Problem — Large Models Can't Run Everywhere

Neural networks used in agriculture must run on cheap smartphones and IoT devices in remote fields — not powerful servers. Standard models are too large and slow. We need a way to make them smaller without losing accuracy.

🌐
02

The Solution — Self-Pruning During Training

PruneVision uses a custom PrunableLinear layer where every weight has a learnable gate. During training, the L1 sparsity loss pushes unnecessary gates to zero — effectively removing those weights. The result is a model that is 30–70% smaller with minimal accuracy loss, ready to run on edge devices.

🧠
03

The Enhancement — Intelligent Preprocessing

Before the pruned model even sees the image, a 4-layer preprocessing pipeline cleans it: quality gating rejects bad photos, CLAHE contrast enhancement normalizes lighting, rembg removes backgrounds, and contour detection isolates only the leaf. This makes the model more accurate on real-world photos.

Why L1 and Not L2 Regularization?

L2

Gradient = 2x — Shrinks large values fast but never pushes small values exactly to zero. Creates small weights, not true sparsity.

L1

Gradient = ±1 — Applies constant pressure regardless of value magnitude. Pushes small values all the way to exactly zero. Creates true sparsity.

L2
never exactly zero
L1
reaches exactly zero ✓

Gate Distribution Analysis

The gate distribution plot reveals two clear clusters:

A large spike near 0 shows gates the network decided were unnecessary — these are the pruned connections. The network learned these weights contributed nothing meaningful to classification and eliminated them.

A smaller cluster near 0.8–1.0 shows gates the network kept fully active — these are essential connections the model protected against the sparsity pressure.

This bimodal distribution is the signature of successful learned sparsity. A failed pruning attempt would show a flat or bell-shaped distribution centered around 0.5.

Why FC1 is Most Pruned: Fully-connected layers have significantly more parameters than convolutional layers. The first fully-connected layer (FC1) after flattening contains 68% sparse gates because it captures redundant feature combinations. Many neuron connections learn similar patterns and can be safely eliminated without degrading accuracy.

Conv Layers Are Selective: Convolutional layers (Conv1, Conv2) are more conservatively pruned (22-31%) because spatial feature extraction is crucial. The network protects filter combinations that detect edges, textures, and disease markers in leaves.

Overall Impact: Despite 39% average sparsity across the network, critical pathway redundancy allows the network to maintain 54% test accuracy — proof that learned sparsity preserves essential computation while eliminating waste.
